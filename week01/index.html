Master Mobile UX

If you design for the smallest, least powerful device first, then you will focus on the content, and your core functionality. That leads to simple, beautiful apps. If you do it the other way around, it'll be like trying to put a marshmallow into a piggy bank, which is neither simple, nor beautiful. -Joel Marsh

This is an important idea for me to think about. I often want to start coding for desktop screens first despite many resources saying to do mobile first. This is especially relevant since mobile use is so prevalent. Do mobile first! I'm basically doing responsive retrofitting when I'm starting from scratch and could easily do mobile first.

It's important to consider that not everyone has access to fast networks. 3G is still very common in some parts of the world.

Consider reachability with one handed grip on mobile devices. It's never occurred to me to put the navigation at the bottom of the screen, but that's where it's easiest for users to access.

Not everyone knows what the hamburger icon means. Putting menu in a border had the most clicks.

Make it simple so people can use is easily. Forms are more mobile-friendly if they fit on one page. Float labels look really good for forms, but are they accessible for screen readers and the like?

Using an anchor tag, you can link your phone number. It makes it easier for people to call from mobile devices.

Ads inside content should be small (unobstrusive) and simple. Stop using pop-ups.
