Object Methods: this

  • To access an object, a method can use the this keyword.
  • The value of "this" is the obejct "before dot," the one used to call the method.
  • The keyword this can be used in any function.
  • The value of this is evaluated during the run-time, depending on the context.
  • Arrow functions have no "this"

Ch5: Objects

  • It's not recommended to use a constructor function to create an empty object. Object literal notation is the better way because it requires less typing and provide a concise way of initializing an object and its properties in one statement.
  • You can access the properties of an object using the dot notation or bracket notation. Dot notation is more common, but bracket notation is good for when you need to access a nonstandard prperty and method names that don't follow naming rules.
  • The new Symbol data type can be used as a computed property key. To be honest, I don't really understand this part yet.
  • You can check if properties or methods exist with the in operator, using !== undefined, or the hasOwnProperty() method.
  • You can loop through all of an object's properties using a for in loop. The Objects.keys() method will return an array of all the keys of any object provided. Object.values() and Object.entries() are other options.
  • Properties don't always appear in the order that they were entered. An object is not an ordered list like an array.
  • Any property can be removed from an object using the delete operator.
  • An object can contain other objects (nesting). The values in nested objects can be accessed by referencing each property name in order using either dot or bracket notation.
  • Objects are copied by reference. This means that if a variable is assigned to an object that already exists, it will simply point to the exact same space in memory. So any changes made using either reference will affect the same object.
  • An object literal can be passed as a parameter to a function. The arguments can be provided by name in any order.
  • The this keyword can be used inside methods to gain access to the object's properties.
  • I'm not sure I understand namespacing yet. I get the basic concept, but the example in the book is confusing.
  • I've done some stuff with JSON, as well as the Math and Date objects.
  • I learned a bit about RegExp last semester, but it was never super clear. Apparently the preferred way to create one is use the literal notation of writing the regular expression between forward slashes.

Ch6: Document Object Model

  • The DOM allows you to access elements of a web page and enable interaction with the page by adding and removing elements, changing the order, content and attributes of elements, and even altering how they are styled.
  • The DOM represents an HTML document as a network of connected nodes. It treats everything on a web page as a node. The HTML tag is the root node. JavaScript can be used to access and modify parts of a web page using the document object.
  • I was wondering why the DOM tree examples had extra text nodes. Apparently it's because the DOM also stores any whitespace that in in the HTML document as text nodes.
  • The getElementById() method returns a reference to the element with a unique id attribute that is given as an argument. getElementsByTagName() will return a live node list of all the elements with the tag name that is provided as an argument. getElementsByClassName() will return a live node list of all elements that have the class name that is supplied as an argument.
  • The document.querySelector() method allows you to use CSS notation to find the first element in the document that matches a CSS selector provided as an argument. So in theory, it can be used instead of document.getElementById() or document.getElementsByClassName(), but it is not a live node list.
  • The getAttribute() method returns the value of the attribute provided as an argument. The setAttribute() method can change the value of an element's attributes. It takes two arguments: the attribute you wish to change, and the new value of the attribute.
  • The classList property is a list of all the classes an element has. The add method can be used to add a class to an element without overwriting any classes that already exist. The remove method will remove a specific class form an element. The toggle method will add a class if an element doesn't have it already, and removes the class if it does have it. The contains method will check to see if an element has a particular class.
  • The innerHTML property returns all the child elements of an element as a string of HTML. It is also writeable and can be used to place a chunk of HTML inside an element. Any code contained with script tags is not executed.
  • Any CSS property names that are separated by dashes must be written in camelCase notation. For example, background-color becomes backgroundColor.

Ch7: Events

  • Event listeners are like setting a notification to alert you when something happens. The event listener will let the program know when the event happens, and the program can then respond appropriately. This allows the program to continue with other tasks while it waits for the event to happen.
  • Inline event handlers are best avoided now that there are better ways.
  • Older event handlers still have the restriction that only one function can be used for each event.
  • Event listeners are the recommended way of dealing with events. The addEventListener() method is called on a node obejct, the node to which the event listener is being applied. It can also be called without a node, in which case it is applied to the global object.
  • Whenever an event handler is triggered by an event, the callback function is called.
  • The target property returns a reference to the node that fired the event.
  • Types of events:
    • Mouse events (click, mousedown, mouseup, dbclick, mouseover, mouseout, mousemove)
    • Keyboard events (keydown, keypress, keyup)
    • Modifier keys (pressing shift, ctrl, alt, and meta)
    • Touch events (touchstart, touchend, touchmove, touchenter, touchleave, touchcancel)
  • An event listener can be removed using the removeEventListener() method.
  • preventDefault() is a method of the event object that can be used inside the callback function to stop the default behavior from happening. Some events do not allow the default behavior to be prevented.
  • When you click on an element, you are actually clicking on all the elements it's nested inside of. An event is said to propagate as it moves from one element to another. Event propagation is the order that the events fire on each element. There are two forms of event propagation: bubbling and capturing. Bubbling is when the event fires on the element clicked on first, then bubbles up the document tree until it reaches the node. Capturing starts by firing an event on the root element, then propagates downwards until it reaches the target element that was clicked on.
  • Event delegation can be used to attach an event listener to a parent element in order to capture events that are triggered by its child elements.
