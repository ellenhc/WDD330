OOP

OOP is a style of programming that involves separating the code into objects that have properties and methods. This allows for encapsulation. Encapsulation keeps the inner working of an object hidden inside with only the essential functionailities being exposed to the end user. OOP also allows for polymorphism, which is that various objects can share the same method. OOP also includes inheritance, which means that we can take an object that already exists and inherit all its properties and methods.

I honestly didn't know until last week that you could do OOP with JavaScript. I'm currently in a Java class right now so I really understand when the textbook says that Java is a "class-based language."

JavaScript contains a nmber of built-in constructor functions such as Object, Array, and Function that can be used to create objects, array, and functions instead of literals. However, it's almost always recommended to use literals for arrays.

By convention, the names of constructor functions or class declarations are capitalized.

The ES6 class declarations are preferable to the constructor function syntax because they are more succint, easier to read, and all code in a class definition is implicitly in strict mode.

The static keyword can be used in class declarations to create a static method. These are called class methods in other languages. Static methods are not available to instances of the class.

Prototypal inheritance means that every class has a prototype property that is shared by every instance of the class. So any properties or methods of a class's prototype can be accessed by every object instantiated by that class.

All classes and constructor functions have a prototype property that returns an object. There are a number of ways to find the prototype of an object: the constructor function's prototype property, Object.getPrototypeOf(), proto property, isPrototypeOf().

It is not possible to overwrite the prototype by assigning it to a new object literal if class declarations are used. It is possible to do this if constructor functions are used, but it sounds problematic. An object instance can overwrite any properties or methods inherited from its prototype by simply assigning a new value to them.

The prototype can be used to add any new properties and methods after the class has been declared. It should be used to define any properties that will remain the same for every instance of the class.

Never use arrays or objects as a default value in prototype.

By default, an object's methods are public in JS. This means they can be queried directly and changed by assignment. But giving users or external services too much access to properties and methods could be dangerous.
