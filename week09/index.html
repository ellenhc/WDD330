Chp 9: The Window Object

The Browser Object Model (BOM) is a collection of properties and methods that contain information about the browser and computer screen. These properties and methods are made available through the window object.

Global variables are created without using the const, let, or var keywords. They can be accessed in all parts of the program.

Global variables are actual properties of a global object. In a browser environment, the global object is the window object. This means that any global variable created is actually a property of the window object.

The window.alert() method pauses the execution of the program and displays a message in a dialog box. The message is provided as an argument to the method.

The window.confirm() method will stop the execution of the program and display a confirmation dialog that shows the mesage provided as an argument, and giving the options of OK or Cancel.

the window.prompt() method will stop the execution of the program. It displays a dialog that shows a messag eprovided as an argument, as well as an input field that allows the user to enter text. This text is then returned as a string when the user clicks OK.

Thw window object has a navigator property that returns a reference to the Navigator object. The Navigator object contains information about the browser being used. *Don't rely on this info though, because it can easily masquerade as a different browser.*

The window.location property is an object that containers info about the URL of the current page.

  • The href property returns the full URL as a string.
  • The protocol property returns a string describing the protocol used.
  • The host property returns a string describing the domain of the current URL and the port number.
  • The hostname property returns a string describing the domain of the current URL.
  • The port property returns a string describing the port number.
  • The pathname property returns a string of the path that follows the domain.
  • The search property returns a string that starts with a '?' followed by the query string parameters.
  • The hash property returns a string that starts with a '#' followed by the fragment identifier.
  • The origin property returns a string that shows the protocol and domain where the current page originated from.
The window.location object also has reload(), assign(), replace(), and toString() methods. The reload() method can be used to force a reload of the current page. The assign() method can be used to load another resource from a URL provided as a parameter. The replace() method is almost the same as the assign() method, except the current page will not be stored in the session history. The toString() method returns a string containing the whole URL.

The window.history property can be used to access info about any previously visited pages inthe current browser session. The window.history.length property shows how many pages have been visited before arriving at the current page. The window.history.go() method can be used to go to a specific page, where 0 is the current page. The window.history.forward() and window.history.back() methods can be used to navigate forwards and backwards by one page respectively.

A new window can be opened using the window.open() method. It takes the URL of the page to be opened as its first parameter, the window title as its second parameter, and a list of attributes as the third parameter.

  • The close() method can be used to close a window.
  • A window can be moved with the window.moveTo() method.
  • The window can be resized with the window.resizeTo() method.

The window.screen object contains info about the screen the browser is displayed on. You can get the height and width of the screen in pixels using the height and width properties. The availHeight and availWidth can be used to find the height and width of the screen. The colorDepth property can be used to find the color bit depth of the user's monitor.

Each window contains a document object that has properties and methods that deal with the page that has been loaded into the window. The write() method simply writes a string of text to the page. If the page has already loaded, it will completely replace the current document. The document.write() method can also be used within a document inside script tags to inject a string into the markup. This will not overwrite the rest of the HTML on the page.

Cookies are small files that are saved locally on a user's computer. Cookies that the form of a text file that contain a list of name/value pairs separated by semicolons. To create a cookie, you assign it to JavaScript's 'cookie jar' using the document.cookie property. This acts like a special type of string. Assigning another cookie to it won't overwrite the entire property, it will just append it to the end of the string. A cookie's value can be changed by reassigning it to document.cookie using the same name but a different value. To see the current contents of the cookie jar, enter document.cookie. Cookies are session cookies by default, meaning they are deleted once a browser session is finished. They can be made persistent by adding "; expires=date" to the end of the cookie when it's set. By default, cookies can only be read by pages inside the same directory and domain as the file was set. The path can be changed so that any page n the root directory can read the cookie by adding the string ; path=/ to the end of the cookie when it is set. Adding the string ; secure to the end of a cookie will ensure itâ€™s only transmitted over a secure HTTPS network. To remove a cookie, you need to set it to expire at a time in the past.

The window.setTimeout() method accepts a callback to a function as its first parameter and a number of milliseconds as its second parameter. Perhaps this can be used to schedule the execution of a function, and repeatedly executing functions at regular intervals. The window.clearTimeout() method can cancel the timeout. The window.setInterval() method will repeatedly invoke the callback function after every given number of milliseconds. The window.clearInterval() method can be used to stop this.

The setTimeout() and setInterval() methods can be used to animate elements on a web page.

Chp 14: HTML5 APIs

The data attribute is a way of embedding data in a web page that is ignored by the browser. They're perfect for adding data that can be used as a hook that the JS program can use to access info about a particular element on the page.

Eaxh element has a dataset property that can be used to access any data attributes it contains.

If a data attribute's name contains hyphens, they are replaced with camel-case notation.

The Web Storage API provides a key-value store on the client's computer that is similar to using cookies, but has few restrictions, more storage capacity, and is easier to use.

The Geolocation API is used to obtain the geographical position of the device.If geolocation is available, it will be a property of the navigator object. Position speed, altitude, heading, and timestamp are the available properties.

Web workers allow processes to be run in the background, adding support for concurrency in JavaScript.

The Service Worker API allows a worker script to run in the background with the added benefit of being able to intercept network requests. This allows you to take alternative action if the network is offline, and effectively create app-like offline experiences.

Websocket is a new protocol that allows two-way communication with a server - AKA push messaging. This means that a connection is kept open and responses are pushed to the client as soon as they are received.

The Notification API allows you to show messages using the system's notifications. This is usually a popup in the corner of the screen, but it changes depending on the operating system.

With HTML5, Flash is no longer needed to add audio and video. An audio clip can be inserted into a page with the audio tag, using the src attribute to point to the audio file. A video can be inserted with the video tag using the src attribute to point to the movie file. Any content inside the tags will only display if the browser does not support them.

The canvas element allows graphics to be drawn onto a webpage in real time using JavaScript. A canvas element is a rectangular element on the web page. It has a coordinate system that starts at (0,0) in the top-left corner. Anything placed inside the tag will only display if the canvas element in unsupported.

Shims and Polyfills are libraries of code that allow you to use APIs as usual because they fill in the necessary code that's not being provided natively by the user's browser. A shim is a piece of code that adds some missing functionality to a browser, although the implementation method may differ slightly from the standard API. A polyfill is a shim that achieves the same functionality while also using the API commands that would be used if the feature was supported natively. With shims and polyfills, your code can use the APIs as normal and it should work as expected in older browsers.
